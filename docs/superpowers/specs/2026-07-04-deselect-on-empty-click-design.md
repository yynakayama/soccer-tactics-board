# 空白クリックで選手選択を解除 — 設計

## 背景 / 問題

選手トークンをクリック/タップすると `selectPlayer()` で選択状態（`state.selected`）になり、
サイドパネルの「選択中」に反映され、トークンがハイライトされる。しかし選択を**解除**する
手段は現状、Escキー（`handleShortcut` の `case "Escape"`）と「選択解除」ボタン
（`#clearSelectionBtn`）の2つだけで、盤面上で完結しない。

ユーザーが期待する自然な操作は「**盤面の何もないところをタップしたら選択が外れる**」。
特にスマホではキーボードが無く、ボタンを探すより空白タップの方が直感的。この近道を追加する。

## 方針

移動モード中に**ピッチ（`#field`）内の空白**をクリック/タップしたら、既存の
`clearSelection()` を呼んで選択を解除する。表示文言の追加はなく、無言の挙動追加のみ。
ゼロ依存・相対パス・i18nの既存規約を維持する。

## スコープ（確定事項）

- **対象モード**: 移動モード（`drawTool === "move"`）のみ。ペン/矢印モードでは空白クリック＝
  描画開始なので解除しない。
- **対象エリア**: ピッチ（`#field`）内のみ。ライン外の余白（`.field-shell`）・サイドパネル・
  ヘッダーは対象外。

## 挙動（既存・変更しない）

- **シングルタップ/クリック**（トークン）→ その選手を選択（`selectPlayer`）。
- **ドラッグ**（トークン）→ 移動。
- **ダブルタップ**（トークン）→ 管理パネルを開く。
- **ペン/矢印モードの空白ドラッグ** → 描画（`startDrawStroke`）。
- Escキー／「選択解除」ボタン → 従来どおり `clearSelection()`。
- 本設計はこれらに一切触れない。

## 変更内容

素のJS 1ファイル（`app.js`）のみ。HTML/CSS/i18n の変更は不要。

### 1. ハンドラ関数を追加（`app.js`）

ピッチ空白クリック時に選択を解除する関数を新設する。ガード条件を**すべて**満たす場合のみ
`clearSelection()` を呼ぶ。

```js
function handleFieldBackgroundClick(event) {
  if (drawTool !== "move") return;              // 移動モードのみ
  if (!state.selected) return;                  // 選択が無ければ何もしない
  if (event.target.closest(".player-token, .ball-token")) return; // コマ/ボールは空白でない
  clearSelection();
}
```

- ガード3条件で、描画・ドラッグ・トークンタップと干渉しない。
- `clearSelection()` は既存関数（`state.selected = null` → `renderSelectionPanel()` +
  `syncSelectedTokens()`）をそのまま再利用。

### 2. リスナー登録（`app.js`、`els.drawLayer.addEventListener("pointerdown", …)` の近く）

```js
els.field.addEventListener("click", handleFieldBackgroundClick);
```

- `click` を使う理由: 本物のタップ/クリック（down→up が同一で移動を伴わない）でのみ発火し、
  ドラッグや描画の途中では発火しないため、余計なガードなしに自然に除外できる。
- 移動モードでは空白は最前面の `.players-layer`（z-index 2 / `pointer-events:auto`）に当たり、
  そのクリックが `#field` にバブリングして届く。`event.target` はトークンでないため
  `closest(".player-token, .ball-token")` は `null` となり解除が走る。
- トークンをクリックした場合は `event.target` がトークン配下なので `closest` がヒットし、
  解除は走らない（選択は `selectPlayer` により維持）。

## なぜこの実装で干渉しないか（根拠）

- ペン/矢印モード: `.draw-layer` が `pointer-events:auto`（最前面 z-index 1 だが移動モードでは
  none）。移動モードでは `.draw-layer` は `pointer-events:none`、`.players-layer` が
  `pointer-events:auto` で最前面。よって空白クリックは移動モードでのみ `#field` に届く。
  加えて `drawTool !== "move"` の早期returnで二重に保護。
- ドラッグ後: 移動を伴うドラッグでは対象がトークンであり、かつ `click` の `event.target` は
  トークン。どちらの観点でも解除されない。

## 触らないもの（非退行）

- i18n 辞書・`data-i18n`・`t()` は**変更なし**（表示文言の追加なし）。
- HTML・CSS は**変更なし**。
- 既存の描画（ペン/矢印）・ドラッグ移動・ダブルタップ・回転ハンドル・Esc/ボタン解除は不変。
- ゼロ依存・ビルド不要・相対パス参照を維持。
- 既存機能（フォーメーション反映／ペン・矢印描画／戦術ガイド／退場／ヘッダー折りたたみ／
  ドロワー／キーボードショートカット／盤面フィット／ダブルタップ）を壊さない。

## アクセシビリティ

- 既存の解除手段（Esc・ボタン）は残るため、キーボード操作の後退はない。
- 追加はポインタ操作の近道であり、支援技術利用者の操作は変わらない。

## 受け入れ基準

1. 移動モードで選手を選択 → ピッチ内の空白をクリック/タップすると選択が解除され、
   サイドパネルが「選択なし」表示に戻る。
2. 移動モードで選手を選択 → 別の選手やボールをクリックしても、その操作の本来の挙動
   （選択切替／ボールは無反応）になり、空白クリック起因の解除は走らない。
3. ペン/矢印モードでピッチの空白をドラッグ/クリックしても選択は解除されず、描画が従来どおり動く。
4. Escキー・「選択解除」ボタンの解除が従来どおり動く。
5. 既存機能（ドラッグ移動・ダブルタップでパネル・回転・戦術ガイド等）が従来どおり動く。

## やらないこと（スコープ外）

- ペン/矢印モードでの空白クリック解除（今回は移動モードのみ）。
- ライン外の余白（`.field-shell`）・パネル・ヘッダーでの解除。
- 解除に伴うヒント表示やトースト等のUI追加。
