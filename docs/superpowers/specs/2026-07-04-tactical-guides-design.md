# 戦術ガイド表示（5レーン・ビエルサライン・3ゾーン）設計

日付: 2026-07-04
ステータス: 承認待ち

## 目的

作戦盤のピッチ上に、戦術説明用のガイドを重ねて表示できるようにする。

1. **5レーン線** — ピッチを幅方向に5分割する境界線4本（点線）
2. **ビエルサライン** — ゴールポストと同じ側のペナルティエリア奥角を結ぶ斜線（各ゴール2本、計4本）
3. **3ゾーン** — ピッチを長さ方向に3等分（ディフェンシブ/ミドル/アタッキングサード）した色分け塗り

それぞれ独立したボタンで表示/非表示を切り替えられ、状態はリロード後も維持される。

## 決定事項（ブレスト結果）

| 論点 | 決定 |
|---|---|
| ビエルサラインの扱い | 5レーンとは別トグルで表示可能にする |
| ビエルサラインの描画範囲 | 両ゴール（計4本） |
| 描画方式 | SVGオーバーレイ1枚（案B）。CSS回転divは不採用 |
| 3ゾーンの配色 | 青（守備）→黄（中盤）→赤（攻撃）のヒート風、薄い半透明 |
| 攻撃方向の前提 | 既存ボードの慣例どおり「味方は左→右へ攻める」。アタッキングサードは右側1/3（縦表示では上側） |

## UI仕様

- ツールバーの「縦表示」ボタンの隣に3つのボタンを追加する:
  - `#toggleLanesBtn` 「5レーン」
  - `#toggleBielsaBtn` 「ビエルサライン」
  - `#toggleThirdsBtn` 「3ゾーン」
- 既存の「縦表示」と同じ `secondary-button` + `aria-pressed` パターン。ラベル文言は固定
- ON時は視覚的に強調する。ガイド用トグルに `is-active` クラスを付与し、背景を `var(--ink)`・文字を白にする（既存の `primary-button` と同等の見た目）
- モバイル幅（680px以下）では既存どおりツールバーが折り返して全ボタンが操作できる

## 状態管理

- `state.guides = { lanes: false, bielsa: false, thirds: false }` を追加（初期値すべてOFF）
- `saveState()` の payload に `guides` を追加
- `loadState()` では `saved?.guides?.lanes === true` のように厳密に boolean 化して復元（欠損・不正値はOFF扱い）。既存の `orientation` の sanitize と同じ流儀

## 描画仕様

### レイヤー構成

- `index.html` の `#field` 内、`#playersLayer` の直前に `<svg id="guidesLayer" class="guides-layer" aria-hidden="true"></svg>` を追加
- CSS: `position: absolute; inset: 0; width: 100%; height: 100%; z-index: 1; pointer-events: none;`
  - 選手レイヤー（z-index: 2）より下、ピッチ背景・CSSマーキングより上に重なる
  - `pointer-events: none` により選手・ボールのドラッグ操作を一切妨げない

### 座標系と縦横対応

- ガイドの形状は**横表示のコンテナ%座標**（x: 長さ方向 0–100、y: 幅方向 0–100）のデータ表として `app.js` に定義する
- ピッチ有効面は既存CSSマーキングと同じ inset 3.5%（3.5〜96.5）
- 縦表示時は既存 `toScreenPosition()` と同じ変換（left = y, top = 100 − x）で各頂点を変換する
- SVG は viewBox を持たず、`<line>` / `<rect>` のジオメトリ属性に **%値を直接指定**する（SVGのジオメトリ属性の%は軸ごとにビューポート寸法基準で解決される）。これによりCSSマーキングと同じ%数値をそのまま使え、縦横どちらのアスペクト比でも追加換算が不要
- viewBox によるスケーリングを使わないため、`stroke-width` や `stroke-dasharray` は実ピクセル値で指定でき、ボードサイズに関わらず線の太さ・点線間隔が一定に保たれる
- `renderGuides()` を新設し、`renderAll()` から呼ぶ。ONのガイドの要素だけを `document.createElementNS` で生成して `replaceChildren` で差し替える

### ガイド形状の定義（横表示の%座標）

**5レーン線（4本の縦断線、x = 3.5 → 96.5）**

| 線 | y |
|---|---|
| 大外(上)/ハーフスペース境界 | 22 |
| ハーフスペース/中央境界(上) | 40.67 |
| 中央/ハーフスペース境界(下) | 59.33 |
| ハーフスペース/大外(下)境界 | 78 |

- 外側2本は既存CSSのペナルティエリア描画（上下 22%〜78%）に一致させる。実寸（68m中40.32m）の比率よりも、盤上でPA角とレーン境界が視覚的に揃うことを優先する
- 中央3レーンは 22〜78 を3等分

**ビエルサライン（4本の斜線分）**

| 線 | 始点（ポスト） | 終点（PA奥角） |
|---|---|---|
| 左ゴール上側 | (3.5, 44) | (19, 22) |
| 左ゴール下側 | (3.5, 56) | (19, 78) |
| 右ゴール上側 | (96.5, 44) | (81, 22) |
| 右ゴール下側 | (96.5, 56) | (81, 78) |

- ポスト位置は既存CSSのゴール描画（上下 44%〜56%）、PA奥角は既存のペナルティエリア描画（奥行き x=19 / 81）に一致させる

**3ゾーン（塗り矩形3枚＋境界線2本）**

| ゾーン | x範囲 | 塗り |
|---|---|---|
| ディフェンシブサード | 3.5〜34.5 | 青 `rgba(31, 99, 209, 0.13)` |
| ミドルサード | 34.5〜65.5 | 黄 `rgba(240, 201, 76, 0.10)` |
| アタッキングサード | 65.5〜96.5 | 赤 `rgba(215, 66, 63, 0.13)` |

- y範囲はいずれも 3.5〜96.5
- 境界線2本（x = 34.5, 65.5）: 白 `rgba(255, 255, 255, 0.35)`、実線、1.5px
- SVG内の描画順: 3ゾーン塗り → 3ゾーン境界線 → 5レーン線 → ビエルサライン（塗りが最背面）

## スタイル

`styles.css` にクラスとして定義し、色・線種はCSS側で管理する。

| クラス | 対象 | スタイル |
|---|---|---|
| `.guide-lane` | 5レーン線 | 白 `rgba(255,255,255,0.55)`、2px、`stroke-dasharray: 2 6`、`stroke-linecap: round`（点線） |
| `.guide-bielsa` | ビエルサライン | ゴールド `rgba(240,201,76,0.85)`、2px、`stroke-dasharray: 8 6`（破線） |
| `.guide-third` | 3ゾーン塗り | 上表の fill、stroke なし |
| `.guide-third-line` | 3ゾーン境界線 | 白 `rgba(255,255,255,0.35)`、1.5px、実線 |

- 5レーン（白点線）とビエルサライン（金破線）で線種と色を変え、同時表示でも判別できる
- ピッチの常設マーキング（白実線・不透明度0.86）より控えめな濃度にし、主従を保つ

## エラー処理

- localStorage の `guides` が欠損・型不正でもすべてOFFで安全に復元する
- SVG生成は静的なデータ表のみを元にするため、ユーザー入力起因の失敗経路はない

## テスト計画

テストフレームワークは導入していないため、Playwright によるブラウザ確認で検証する。

1. 3つのトグルそれぞれのON/OFFで該当ガイドのみが表示/非表示になること
2. ON状態でリロードしても表示状態とボタンの押下表示が復元されること
3. 縦表示⇔横表示の切り替えでガイドが正しく追従すること（レーン外側線がPA角と、ビエルサライン端点がポスト・PA奥角と一致）
4. ガイド表示中も選手・ボールをドラッグできること（pointer-events: none の確認）
5. 3ゾーンの向き: 横表示で右側、縦表示で上側がアタッキングサード（赤）であること

## 対象外（YAGNI）

- ニアゾーン（ポケット）の塗り、3ゾーン境界のカスタマイズ、攻撃方向の反転設定は今回実装しない（SVGデータ表への追加で将来対応可能）
