# サッカーボード 縦/横表示の切り替え 設計

日付: 2026-06-20

## 背景・目的

現状のボードは横向き（ランドスケープ、アスペクト比 105:68）固定。スマホを縦に持ったときや、縦長でピッチ全体を見たいときのために、**縦表示にも切り替えられる**ようにする。

選手・ボールの座標データの意味は変えず、描画と当たり判定だけを向きに応じて変換する（座標は1セットのまま、向きはビューの問題として扱う）。

## 決定事項（ブレストの結論）

1. **切り替え方法**: ツールバーの手動ボタン。選択は保存。（自動回転はしない）
2. **縦の向き**: 味方（青）が下、相手（赤）が上。自陣ゴールが下で、上に向かって攻める。
3. **実装方針**: 座標は今のまま保持し、描画時に向きを変換（案1）。
   - フィールド全体のCSS回転（案2）や、縦横で座標を二重管理（案3）は不採用。

## 座標モデル（不変）

- `x`: 長辺方向（ゴール→ゴール）。味方は自陣ゴール側 x≈8、相手ゴール側 x≈92。0〜100。
- `y`: 短辺方向（フィールドの幅）。0〜100。
- 既存の `FORMATIONS`・保存データはそのまま流用。データ移行は不要。

## 設計詳細

### 1. 状態と永続化

- `state.orientation` を追加（`"horizontal"` | `"vertical"`、デフォルト `"horizontal"`）。
- `saveState` の payload に `orientation` を含める。
- `loadState` で読み込み、`"horizontal"` / `"vertical"` 以外は `"horizontal"` にフォールバック。

### 2. UI（ツールバー）

- `index.html` のツールバー（配置セレクトの並び）に切り替えボタンを1つ追加。
  - 例: id=`toggleOrientationBtn`、`secondary-button` スタイル。
  - ラベルは「次に切り替わる向き」を表示。横のとき「縦表示」、縦のとき「横表示」。
- クリックで `orientation` を反転 → `saveState()` → `renderAll()`（ラベルも更新）。

### 3. 座標変換（描画＆ドラッグ）

データ座標 ⇔ 画面位置(%) の変換を専用ヘルパーに集約し、トークン生成時・ドラッグ中の両方で使う。

- **横（horizontal）**: `left = x%`, `top = y%`（現状どおり）
- **縦（vertical, 味方下・相手上）**: `left = y%`, `top = (100 − x)%`
  - 自陣ゴール(x≈8)が下(top≈92%)、相手ゴール(x≈92)が上(top≈8%)。左右反転は起きない（横画像を反時計回り90°回転したのと同じ向き）。

ドラッグの逆変換（フィールド矩形 W×H、ポインタ相対座標 px,py）:

- 横: `x = px/W*100`, `y = py/H*100`
- 縦: `y = px/W*100`, `x = 100 − py/H*100`

実装メモ:

- トークンの left/top 設定を `setTokenPosition(token, team, player)` のような関数に切り出し、`createPlayerToken` / `createBallToken` / `moveDrag` から共通利用する。
- 番号・名前ラベルは要素自体を回転しないため常に正立のまま。
- `clampX` / `clampY` はデータ座標に対する制限なので変更不要（変換はその外側で行う）。

### 4. ピッチのライン（CSS）

- フィールド要素 `#field` に `vertical` クラスを付け外し（`renderAll` または専用の `applyOrientation()` で）。
- アスペクト比: 横 `105 / 68`、縦 `68 / 105`。
- `.field.vertical` 用のCSSで各マーキングを縦向きに再配置:
  - ハーフライン: 水平の線（`top: 50%`、左右いっぱい）。
  - ペナルティエリア / ゴールエリア / ゴール: 上端・下端に配置（既存の left/right 要素を top/bottom に読み替えて再配置）。
  - センターサークル / センタースポット: 中央のまま。
- 芝のストライプ（`repeating-linear-gradient`）の向きを縦に合わせる（任意・見た目のみ）。

### 5. レイアウト/サイズ

- 縦のフィールドは縦長になるため、デスクトップで間延びしないよう `.field-shell`（または `.field`）に**最大幅を設定して中央寄せ**する。
- スマホ縦では従来どおり画面幅いっぱいに表示。

## 影響範囲

- `app.js`: state 追加、save/load、ボタンのイベント、座標変換ヘルパー、`createPlayerToken`/`createBallToken`/`moveDrag` の位置設定、orientation 適用。
- `index.html`: ツールバーに切り替えボタン追加。
- `styles.css`: `.field.vertical` のマーキング群、アスペクト比、縦時の最大幅・中央寄せ。

## 非対象（YAGNI）

- 画面の向きに応じた自動回転。
- 縦/横で別々の選手配置の保存。
- 任意角度の回転やズーム。

## 確認/テスト観点

- 横→縦→横と切り替えて、選手・ボールの相対位置が保たれる（向きだけ変わる）。
- 縦で味方が下・相手が上になっている。自陣ゴールが下。
- 縦でドラッグした位置が正しく追従し、再読み込み後も保持される（orientation も保存される）。
- 縦でピッチのライン（ハーフライン水平、PA・ゴールが上下）が正しく描かれる。
- スマホ幅・デスクトップ幅の両方でフィールドが破綻なく収まる。
