# ヘッダー／側面パネルのしまい込み ＋ タブレット最適化 — 設計

作成日: 2026-07-04

## 背景・目的

Soccer Board（戦術ホワイトボード）は、画面上部に固定ヘッダー（`.app-header`：ブランド＋
配置/表示切替/リセットのツールバー）を持ち、右に管理用の側面パネル（`.side-panel`）を
常時表示している。タブレット横向きでは、この**上のヘッダー**と**右の側面パネル**が
フィールド（盤面）を狭め、ボードを大きく使いたい場面で邪魔になる。

本作業のゴールは **盤面を最大化できるようにする** こと。具体的には:

1. ヘッダーを手動で折りたたんで細いバー1本にできる。
2. 側面パネルをかぶせ型ドロワー化し、既定は閉じておき、選手選択時だけ自動で開く。

「選手を選択したとき以外はパネルを閉じていていい」というユーザーの利用実態に合わせる。

## スコープ

- ヘッダーの折りたたみ（トグル、状態保存）。
- 側面パネルのかぶせ型ドロワー化（トグル、スクリム、選択連動の自動開閉）。
- タッチ操作の軽微な配慮（タッチ領域・アニメーション）。

### 非スコープ（YAGNI）

- 選手トークンや方向ハンドルのサイズ拡大（今回ユーザーが課題として挙げていない）。
- ピンチ拡大・スクロールの誤作動対策（同上）。
- ボトムシート等のレイアウト大改造。
- ヘッダー内ツールバーのメニュー集約（別案として不採用）。

## 全体方針

採用案 = **A案：独立ドロワー ＋ 選択で自動開く**（かぶせ型オーバーレイ、全画面幅で統一）。

- ヘッダーとパネルはそれぞれ独立して開閉できる。
- パネルは右からスライドインする「かぶせ型」ドロワー。開いても盤面はリフローせず、
  上に重なる。自動開閉が頻繁でも盤面が伸縮せずガタつかない。
- デスクトップ幅も含め挙動を統一（実装・テストが単純、挙動が一貫）。
  - トレードオフ: デスクトップの「横並び常時表示」は廃止。開いている間はパネルが
    盤面の右側 約360px に重なる。ユーザーの利用実態（既定は閉、選択時のみ開）に
    合致するため許容する。

## 機能仕様

### ① ヘッダーの折りたたみ

- `.app-header` のブランド横に**折りたたみトグルボタン**（chevron ▾/▸、`#toggleHeaderBtn`）を追加。
- 状態は2つ:
  - **展開**（既定）: 現状どおりブランド＋ツールバー全表示。
  - **折りたたみ**: 細いバー1本。ブランドマーク「SB」＋展開ボタンのみ表示。ツールバー
    （配置select・反映・縦表示・5レーン・ビエルサ・3ゾーン・ボードリセット）は非表示。
- 折りたたみは `.app-header` に `is-collapsed` クラスを付与し、CSSでツールバーを
  `display: none`、ヘッダーの余白・高さを縮小して実現。
- 盤面内の描画ツールバー（`.draw-toolbar`：移動/ペン/矢印・色・戻す・全消去）は
  ヘッダー外なので折りたたみの影響を受けず**残る**。折りたたみ中も基本操作は維持される。
- ボードリセットはヘッダーに隠れるが、キーボードショートカット `R` は引き続き有効。
- ボタンは `aria-expanded` / `aria-controls` を持ち、ラベルは状態に応じて更新
  （既存 `applyOrientation()` と同じ「クラス付け替え＋aria更新」パターン）。

### ② 側面パネルのかぶせ型ドロワー

- `.side-panel` を、右端にドッキングした**オーバーレイドロワー**にする。
  - `position: fixed`（または `absolute`）で右端に配置、幅は現状のパネル幅（約 320〜380px）を踏襲。
  - 既定は画面外（`transform: translateX(100%)`）に退避＝閉。開く時 `translateX(0)`。
  - `transition` でスライド。閉じている時は `pointer-events: none` で盤面操作を邪魔しない。
- **手動トグル**: 右端に常設の開閉タブ／ボタン（`#togglePanelBtn`、ラベル「パネル」）。
  タップで開閉をトグル。
- **スクリム**: ドロワーが開いている間、背後にうっすら半透明のスクリム（`.panel-scrim`）。
  スクリムのタップで閉じる。スクリムは閉時 `pointer-events: none` かつ不可視。
- パネルの中身（選択中／交代／退場／味方登録／相手登録／メモ）は現状のまま。ドロワー内は
  縦スクロール可能（`overflow-y: auto`）。

### ③ 選択連動の自動開閉

> **実装後修正（2026-07-04）**: 当初は「選択で自動オープン」だったが、ドラッグ移動の
> たびにパネルが開いて邪魔という実機フィードバックを受け、**ダブルタップで開く**方式に
> 変更した（下記が現行仕様）。

パネルの開閉状態は単一のフラグ `panelOpen`（真偽値）で管理する。状態遷移:

- **手動トグル**（`#togglePanelBtn` / スクリムタップ）: `panelOpen` を反転。
- **シングルタップ / ドラッグ / 矢印キー移動**: 選択・移動のみ。パネルは開かない
  （`selectPlayer()` はパネルを操作しない）。
- **同一選手のダブルタップ**（400ms以内に2回、移動を伴わないタップ。`endDrag` の
  `registerTokenTap()` で判定）: `panelOpen = true`（オープン）＋ドロワー先頭へスクロール。
- **`clearSelection()`**: `panelOpen = false`（強制クローズ）。
  - Esc・解除ボタン・盤面背景タップはいずれも既存の `clearSelection()` を経由するため、
    自動的にこのクローズが効く（「解除で常に閉じる」）。
- 手動で開いた（無選択で `panelOpen=true` にした）場合は、ロスター管理・メモ作業のため
  開いたままキープされる。ただしその最中に選手を選択→解除すると、上記ルールにより閉じる
  （ユーザー選択どおりの割り切り）。

`panelOpen` を反映する適用関数 `applyPanelOpen()` を用意し、`.side-panel` の
`is-open` クラス、スクリムの表示、トグルボタンの `aria-expanded` を一括更新する。

### ④ タブレットのタッチ配慮

- トグルボタン・スクリムの当たり判定は最低 44px 四方を確保。
- スライドアニメは `transition`。`@media (prefers-reduced-motion: reduce)` 時はアニメ無効。
- ドロワーが閉じている間は `pointer-events: none`、スクリムも同様で、盤面のドラッグ・
  描画を一切邪魔しない。

## 実装ポイント（既存コードへの接続）

### index.html

- `.app-header` 内にヘッダー折りたたみトグル `#toggleHeaderBtn` を追加。
- `main.layout` 直下（または `.app` 直下）に、パネルのトグルボタン `#togglePanelBtn` と
  スクリム要素 `.panel-scrim` を追加。`.side-panel` はそのまま流用。

### styles.css

- `.app-header.is-collapsed` でツールバー非表示・高さ圧縮。
- `.side-panel` をオーバーレイドロワー化（`fixed`/`transform`/`transition`）。
  `.side-panel.is-open` で表示。既存の `.layout` グリッド（右カラム）からパネルを外し、
  盤面が常に全幅を使えるようにする。
- `.panel-scrim` の表示/非表示、`pointer-events` 制御。
- 既存メディアクエリ（`max-width: 1020px` / `680px`）を、ドロワー化に合わせて整理。
- `@media (prefers-reduced-motion: reduce)` を追加。
- **実装後修正（2026-07-04）**: 盤面サイズは CSS の固定見積り（`100vh - 200px`）では
  なく、`app.js` の `fitBoardSize()` がヘッダー等の実測高さから残り高さを計算して
  `.board-area` の `max-width` をインラインで与える方式に変更。`.board-area` には
  `width: 100%` を明示（`margin-inline: auto` により grid の stretch が外れて内容幅に
  縮むのを防ぐ）。縦表示は既存の `:has()` ルールに委譲。
- **他セッションとの衝突回避**: 本作業の CSS は既存ルールの書き換えを避け、
  `styles.css` 末尾への追記（append-only）で後勝ち上書きしている。

### app.js

- `els` に `toggleHeaderBtn` / `togglePanelBtn` / `panelScrim` を追加。
- `state` に `ui: { headerCollapsed: false }` を追加（`panelOpen` は選択連動の
  一時状態のため、モジュールスコープ変数 `let panelOpen = false` で保持し、保存はしない）。
- `loadState()`: `saved?.ui?.headerCollapsed` を読み込む。リロード時 `state.selected` は
  既存どおり null リセットされるため、`panelOpen` は既定 false（閉）で開始。
- `saveState()`: payload に `ui: { headerCollapsed: state.ui.headerCollapsed }` を相乗り。
- `bindEvents()`:
  - `toggleHeaderBtn` クリックで `state.ui.headerCollapsed` を反転→ `applyHeaderCollapsed()` ＋ `saveState()`。
  - `togglePanelBtn` / `panelScrim` クリックで `panelOpen` を反転→ `applyPanelOpen()`。
- `selectPlayer()` の末尾で `panelOpen = true; applyPanelOpen();`。
- `clearSelection()` の末尾で `panelOpen = false; applyPanelOpen();`。
- 新規: `applyHeaderCollapsed()`（`.is-collapsed` クラス＋aria＋ラベル更新）、
  `applyPanelOpen()`（`.is-open` クラス・スクリム・aria更新・必要ならスクロール）。
- `renderAll()`（または `init()`）から `applyHeaderCollapsed()` / `applyPanelOpen()` を呼び、
  初期状態を反映。

## 状態・永続化

- 永続化するのは `ui.headerCollapsed` のみ（既存 `localStorage` payload に相乗り）。
- `panelOpen` は非永続（選択連動の一時状態）。リロード時は選択なし＝閉で開始。

## テスト方針

手動確認（Playwright でタブレット実寸のスクリーンショット検証）:

- ビューポート: タブレット横 1024×768、タブレット縦 820×1180。
- ヘッダー折りたたみ→展開が効き、細いバー表示になる。折りたたみ中も描画ツールバーが残る。
- パネルのトグル開閉、スクリムタップで閉じる。
- 選手トークン／ロスター行をタップ→パネルが自動オープン＋選択中表示。
- Esc・解除ボタン・盤面背景タップ→パネルが自動クローズ。
- リロード後、ヘッダー折りたたみ状態が復元され、パネルは閉で開始。
- reduced-motion 設定時にスライドアニメが無効。

## エッジケース

- ヘッダー折りたたみとパネル開閉は独立。両方同時でも破綻しない。
- パネルを手動オープン中に選手を選択→解除すると閉じる（ルールどおり、許容）。
- `localStorage` がブロックされていても盤面は動作（既存の try/catch を踏襲）。
- 旧ブラウザで `:has()` 等に依存しない実装にする（クラス付与ベース）。
