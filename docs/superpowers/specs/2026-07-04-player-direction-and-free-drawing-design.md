# サッカーボード 選手の向き表示＋フリー描画 設計

日付: 2026-07-04

## 背景・目的

作戦説明で「選手がどちらを向いているか（体の向き）」と「動き出しやパスコース」を伝えたい。

1. 選手コマに**向き**を持たせ、**背中側に弧（三日月マーク）**を表示して向きが一目でわかるようにする。
2. ピッチ上に**フリーハンドの線と矢印**を4色で描けるようにする。

## 決定事項（ブレストの結論）

1. **向きの操作**: 選択中のコマに**回転ハンドル**を表示し、ドラッグで回す。（キー操作・自動追従はしない）
2. **線種**: フリーハンド（ペン）と、終点に三角の頭が付く**矢印**の2ツール。直線ツールはなし。
3. **色**: 黄・赤・青・白の4色から選択。
4. **描画技術**: SVGレイヤー（Canvasは不採用。リサイズ・縦横切替時の再描画や高DPI対応の手間が大きいため）。

## データモデル

### 選手の向き

- 味方・相手の選手オブジェクトに `dir`（度数、数値）を追加。ボールには持たせない。
- **ピッチ座標系**の角度: 0°＝+x方向（相手ゴール方向、横表示の右向き）、時計回りに増加（yは下向きが正）。0〜360に正規化。
- 初期値: 味方 `0`、相手 `180`（互いに向き合う）。
- 保存: 選手オブジェクトの一部として localStorage に入る。読み込み時、`dir` が数値でなければチームごとの初期値で補完（**既存の保存データはそのまま動く**）。

### 線

- `state.drawings`: 配列。1本のストローク＝

  ```js
  { id, tool: "pen" | "arrow", color: "yellow" | "red" | "blue" | "white", points: [{ x, y }, ...] }
  ```

- 座標は**ピッチ座標（0〜100）**。縦横どちらの表示で描いても同じデータになる。
- `saveState` の payload に `drawings` を追加。読み込み時に検証（下記エラー処理）。
- ツール・選択色・描画中ストロークは**実行時状態のみ**（保存しない。リロード後は「移動」モード・黄に戻る）。

## 設計詳細

### 1. 向きの表示（CSS）

- コマの円（`.player-token .number`）の縁の**背中側**（`dir` の反対側）に、チームカラーより濃い色の**半円弧**を重ねる。
- 実装: トークン内に弧用の要素を1つ追加し、`--dir-screen`（画面角度）のCSS変数で `transform: rotate()`。弧自体は border による半円リングなどCSSのみで描く。
- 画面角度への変換: 横表示 `dir` そのまま、縦表示 `dir − 90°`（既存の `toScreenPosition` と同じ回転。ピッチ +x が画面上方向になるため）。
- 背番号・名前タグは回転しない（正立のまま）。

### 2. 回転ハンドル

- **選択中の選手コマのみ**、円の中心から向いている方向へ少し離れた位置（円の外周＋約10px）に小さな丸ハンドルを表示。位置はCSSで `rotate(var(--dir-screen)) translateX(距離)`。
- ハンドルを pointerdown → ドラッグで、**コマ中心→ポインタの角度**（`atan2`）に向きが追従。縦表示時は +90° してピッチ角度に戻す。
- ハンドルのイベントは `stopPropagation` で本体の移動ドラッグと分離。pointerup で `saveState()`。
- 味方・相手どちらも回転可。ボールは対象外。

### 3. 描画レイヤー（SVG）

- `#field` 内、マーキングと `#playersLayer` の**間**に `<svg id="drawLayer">` を追加。線は**選手コマの下**に描かれ、コマや背番号が隠れない。
- SVGの `viewBox` はフィールドの実ピクセルサイズ（`0 0 w h`）。描画時に各点をピッチ座標 → `toScreenPosition` → ピクセルに変換。ウィンドウリサイズ時と `renderAll` 時に再描画（既存の全再レンダー方式と同じ）。
- ストローク: `<path>`、線幅約3px、`stroke-linecap/linejoin: round`。矢印は終端の最後の線分方向から三角形 `<polygon>` を計算して付ける（SVG marker は色ごとの定義が要るため使わない）。
- なぞり中の間引き: 直前の点からピッチ座標で約0.8以上動いたときだけ点を追加。

### 4. ツールバーとモード

- `.board-area` 内、スコア帯とフィールドの間に描画ツールバーを追加:
  **[移動] [ペン] [矢印] ｜ 色スウォッチ×4（黄・赤・青・白） ｜ [1つ戻す] [全消去]**
- モードは `移動`（デフォルト）/ `ペン` / `矢印` の排他。ボタンは `aria-pressed` で状態表示。
- **移動モード**: 今まで通り（選手・ボールをドラッグ、線の上も操作可。`#drawLayer` は `pointer-events: none`）。
- **ペン/矢印モード**: `#field` に `draw-mode` クラスを付け、`.players-layer` を `pointer-events: none` に、`#drawLayer` を `pointer-events: auto` に。カーソルは crosshair。pointerdown で `setPointerCapture` して描き始め、pointerup で確定。点が2つ未満なら破棄。
- 色スウォッチは新しく描く線にだけ効く（描いた線の色は変えられない）。
- **1つ戻す**: `drawings` の末尾を1本削除（redoなし。空なら何もしない）。
- **全消去**: `window.confirm` で確認してから全削除（既存の破壊的操作と同じ流儀）。

### 5. 縦横切替・リセットとの整合

- 線・向きともピッチ座標で保持し描画時に変換するため、**縦表示に切り替えると線も向きも一緒に回る**。
- **ボードリセット**: 線を全消去し、向きを初期値（味方0°・相手180°）に戻す。
- **反映**（フォーメーション適用）: 味方の向きを 0° に戻す。**相手登録のリセット**: 相手の向きを 180° に戻す（配置し直し＝仕切り直しの扱い）。線はどちらも維持。

## エラー処理

- 読み込み時に `drawings` を検証: 配列でなければ空に。各ストロークは `tool`/`color` が既知の値、`points` が2点以上の配列であることを確認し、不正なものは除外。点は 0〜100 にクランプ。上限はストローク300本・1本600点（超過分は切り捨て）。
- localStorage が使えなくても描画・回転は動作継続（既存と同じ方針）。

## 影響範囲

- `app.js`: `dir` の初期値・サニタイズ、回転ハンドルのドラッグ処理、`drawings` の state/save/load、描画モードとポインタ処理、SVG再描画、ツールバーのイベント、リセット系への組み込み。
- `index.html`: 描画ツールバー、`#drawLayer` の追加。
- `styles.css`: 向きの弧・回転ハンドル、ツールバー、`draw-mode` 時の pointer-events・カーソル。

## 非対象（YAGNI）

- 直線ツール・図形ツール・テキスト書き込み。
- 描いた線の選択・移動・色変更・個別削除（消しゴム）。redo。
- キー操作での回転、移動方向への自動向き変更、向きのスナップ。
- 線・ツール選択のエクスポートや画像保存。

## 確認/テスト観点

テスト基盤がないため、dev-server 起動＋Playwright での表示・操作確認と手動チェック:

- 選手を選択→ハンドルをドラッグして回る。背中側の弧が向きに追従する。リロード後も向きが保持される。
- 縦表示に切り替えると向きの見た目も90°回る。縦表示のままでも回転操作が正しい。
- ペンで描く→矢印で描く→色を変えて描く。線が選手コマの下に描かれる。
- 縦で描いた線が横に切り替えても正しい位置・形になる。リロード後も線が残る。
- 1つ戻す／全消去（確認ダイアログ）が効く。
- 移動モードに戻すと選手ドラッグ・キー移動が今まで通り動く。描画モード中はポインタで選手が動かない（キー移動は対象外）。
- 既存の保存データ（`dir`・`drawings` なし）で開いても壊れない。
